# Fix: Tasks page missing sidebar, broken buttons and actions

## Root Cause Analysis

Three separate issues are causing the problems:

### Issue 1: Missing `workspaces` in shared Inertia page props
`HandleInertiaRequests` shares `currentWorkspace` (singular) but NOT `workspaces` (plural). The `AppSidebar.vue` reads `page.props.workspaces` for the `WorkspaceSwitcher` component, which gets `undefined` → `[]`. The sidebar still renders (it's a plain `<aside>`), but the workspace switcher is empty.

**File**: `app/Http/Middleware/HandleInertiaRequests.php` (line 27)

### Issue 2: Missing `projects` in shared Inertia page props  
`HandleInertiaRequests` does NOT share `projects`. The `AppSidebar.vue` reads `page.props.projects` for the project list at the bottom of the sidebar. Only the `TodoController::index` passes `projects` as a page prop. On other pages (Dashboard, Calendar, Activity, Notifications, Projects index), the sidebar shows no projects.

**File**: `app/Http/Middleware/HandleInertiaRequests.php`

### Issue 3: Workspace switch route name mismatch
`WorkspaceSwitcher.vue` calls `route('workspaces.switch', workspace.id)`, but Wayfinder generates the export as `switchMethod` (because `switch` is a reserved word in JS). The custom route helper in `resources/js/lib/route.ts` registers it as `workspaces.switchMethod`, NOT `workspaces.switch`. So the lookup fails and falls back to a console warning + incorrect URL `/workspaces/switch` (missing the workspace ID).

**Files**: 
- `resources/js/components/workspace/WorkspaceSwitcher.vue` (line 28)
- `resources/js/routes/workspaces/index.ts` (export is `switchMethod`)
- `resources/js/lib/route.ts` (route name registration)

### Issue 4: Route helper fallback produces broken URLs
When the route helper can't find a route name, it returns `/${name.replace('.', '/')}` which strips the dot and adds a leading slash. For `route('workspaces.switch')` this produces `/workspaces/switch` — no workspace ID substituted. Many button clicks silently fail because the POST/PUT goes to the wrong URL.

**File**: `resources/js/lib/route.ts` (lines 56-63)

## Implementation Plan

### Step 1: Share `workspaces` and `projects` in HandleInertiaRequests
**File**: `app/Http/Middleware/HandleInertiaRequests.php`

Add to the `share()` return array:
```php
'workspaces' => fn () => $user
    ? $user->workspaces()->get()
    : [],
'projects' => fn () => $user
    ? $user->currentWorkspace()?->projects()->active()->get() ?? collect()
    : collect(),
```

### Step 2: Fix WorkspaceSwitcher route name
**File**: `resources/js/components/workspace/WorkspaceSwitcher.vue` (line 28)

Change:
```ts
route('workspaces.switch', workspace.id),
```
To:
```ts
route('workspaces.switchMethod', workspace.id),
```

### Step 3: Verify all route names used across the app match Wayfinder exports
Check every `route('...')` call against the actual Wayfinder export names. The Wayfinder convention renames reserved words (e.g., `switch` → `switchMethod`).

**Files to audit**:
- `resources/js/pages/tasks/Index.vue` — uses `todos.index`, `todos.complete`, `todos.uncomplete`, `todos.destroy`, `todos.show`
- `resources/js/pages/tasks/Show.vue` — uses `todos.complete`, `todos.uncomplete`, `todos.update`, `comments.store`, `checklists.store`, `checklistItems.store`, `checklistItems.toggle`
- `resources/js/components/task/TaskCreateDialog.vue` — uses `todos.store`
- `resources/js/components/task/TaskDetail.vue` — uses `todos.update`, `todos.complete`, `todos.uncomplete`, `comments.store`, `checklists.store`, `checklistItems.store`, `checklistItems.toggle`
- `resources/js/components/AppSidebar.vue` — uses `projects.show`
- `resources/js/components/workspace/WorkspaceSwitcher.vue` — uses `workspaces.switchMethod` (fixed in step 2)
