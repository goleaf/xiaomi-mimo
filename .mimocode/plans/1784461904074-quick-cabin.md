# Plan: Comprehensive Database Seeder

## Context

The `/projects` page has data (3 projects, 12 todos exist), but all other relations are empty — no labels, tags, checklists, comments, reminders, or activity logs. The user wants maximum relations seeded so every UI component has data to display.

## What to change

**Single file:** `database/seeders/DatabaseSeeder.php`

## Seed Data Plan

The seeder will create rich, interconnected data for all 14 models:

### Users (3)
1. **Demo User** (owner) — `demo@example.com`
2. **Alice Chen** (admin) — `alice@example.com`
3. **Bob Smith** (member) — `bob@example.com`

### User Preferences (3)
- One per user with varied settings (theme, timezone, language, etc.)

### Workspace (1)
- "Acme Projects" with all 3 users as members (owner, admin, member roles)

### Projects (5)
1. Product Launch — `#6366f1` rocket (5 todos)
2. Website Redesign — `#ec4899` palette (4 todos)
3. Mobile App — `#14b8a6` smartphone (4 todos)
4. Marketing Campaign — `#f59e0b` megaphone (3 todos)
5. DevOps Infrastructure — `#ef4444` server (2 todos)
6. 1 archived project: "Legacy System" — `#6b7280` archive (2 todos)

### Todos (20+)
Mix of all statuses, priorities, dates, and features:
- **In Progress** (5): with descriptions, due dates, assigned users
- **Pending** (8): various priorities, some with subtasks
- **Completed** (5): with `completed_at` timestamps
- Some with `is_pinned`, `is_favorite`
- Some with `start_date`, `estimated_time`, `spent_time`
- Some overdue (past due date)
- Subtasks (parent_id pointing to other todos)

### Labels (6)
- "Bug", "Feature", "Enhancement", "Documentation", "Testing", "Security"
- Each workspace-scoped, with distinct colors
- Attached to todos via `todo_label` pivot

### Tags (5)
- "frontend", "backend", "design", "urgent", "review"
- Each workspace-scoped
- Attached to todos via `todo_tag` pivot

### Checklists (4)
- 2-3 per selected todo, with descriptive names
- Each with 3-4 ChecklistItems (mix of checked/unchecked)

### Comments (8)
- On various todos, from different users
- Realistic comment text

### Reminders (4)
- On selected todos, for different users
- Mix of `in_app`, `email`, `browser` types
- Some sent, some pending

### Activity Logs (10)
- Recent activity events across workspace
- Created, updated, completed, archived events
- Polymorphic to various subjects

### Notifications (5)
- Laravel notifications for the demo user
- Mix of read/unread

## Files to Create/Modify

1. **`database/seeders/DatabaseSeeder.php`** — Complete rewrite with all seed data
2. **`database/seeders/UserSeeder.php`** — New: users + preferences
3. **`database/seeders/WorkspaceSeeder.php`** — New: workspace + members
4. **`database/seeders/ProjectSeeder.php`** — New: projects
5. **`database/seeders/TodoSeeder.php`** — New: todos + subtasks
6. **`database/seeders/LabelSeeder.php`** — New: labels + todo_label pivots
7. **`database/seeders/TagSeeder.php`** — New: tags + todo_tag pivots
8. **`database/seeders/ChecklistSeeder.php`** — New: checklists + items
9. **`database/seeders/CommentSeeder.php`** — New: comments
10. **`database/seeders/ReminderSeeder.php`** — New: reminders
11. **`database/seeders/ActivityLogSeeder.php`** — New: activity logs

The `DatabaseSeeder` will orchestrate all sub-seeders in correct dependency order.

## Execution

1. Create all seeder files
2. Run `php artisan migrate:fresh --seed` to rebuild DB with full seed data
3. Verify `/projects` page shows projects with todos_count
4. Verify project detail pages show todos with labels, tags, checklists
5. Run `vendor/bin/pint --dirty --format agent`
