# Todo SaaS Application — Full Implementation Plan

## Current State

Fresh Laravel 13 + Inertia v3 + Vue 3 scaffold. SQLite by default. shadcn-vue (22 components). Fortify auth (login, register, 2FA, passkeys). No business logic.

**Stack (confirmed):** Laravel 13, PHP 8.3+, Inertia v3, Vue 3, Pinia, shadcn-vue, Tailwind CSS 4, SQLite3 (WAL), Sanctum, Vite. No PWA yet.

## Phase 1: Infrastructure & Database

### 1.1 SQLite WAL Configuration

**File:** `config/database.php`
- Set `journal_mode` → `'WAL'`
- Set `busy_timeout` → `5000`
- Set `synchronous` → `'NORMAL'`
- Set `foreign_key_constraints` → `true`
- Add `cache_size` via `pragmas` array

**File:** `app/Providers/AppServiceProvider.php`
- Add `DB::afterCreating` or connection event to run `PRAGMA temp_store = MEMORY` and `PRAGMA cache_size = -64000`

### 1.2 UUID Infrastructure

**New file:** `app/Concerns/HasUuid.php` — trait that overrides `newUniqueId()`, `getIncrementing()`, `getKeyType()`. Uses Laravel's built-in `HasUuids` trait from `Illuminate\Database\Eloquent\Concerns\HasUuids`.

**Migration:** Modify `0001_01_01_000000_create_users_table.php`
- Change `$table->id()` → `$table->uuid('id')->primary()`
- Update password_reset_tokens and sessions foreign keys

### 1.3 All Migrations (new files in `database/migrations/`)

Tables in dependency order:

1. **users** (modify existing) — uuid PK, name, email, email_verified_at, password, remember_token, timestamps
2. **password_reset_tokens** (modify existing)
3. **sessions** (modify existing)
4. **cache, cache_locks** (keep existing)
5. **jobs, job_batches, failed_jobs** (keep existing)
6. **personal_access_tokens** (Sanctum — `php artisan sanctum:install`)
7. **workspaces** — uuid PK, name, slug, description, owner_id (FK users), timestamps
8. **workspace_members** — uuid PK, workspace_id (FK), user_id (FK), role (enum: owner/admin/member), timestamps. Unique constraint on [workspace_id, user_id]
9. **projects** — uuid PK, workspace_id (FK), name, description, color, icon, is_archived (default false), position (int for sorting), timestamps
10. **todos** — uuid PK, project_id (FK nullable), workspace_id (FK), assigned_to (FK users nullable), parent_id (FK todos nullable for subtasks), title, description (text nullable), status (enum: pending/in_progress/completed), priority (enum: none/low/medium/high/urgent), due_date (date nullable), start_date (date nullable), estimated_time (int minutes nullable), spent_time (int minutes nullable), is_pinned (bool), is_favorite (bool), is_archived (bool), is_recurring (bool), recurring_rule (string nullable — iCal RRULE), position (int for manual sort), completed_at (timestamp nullable), timestamps, soft_deletes
11. **checklists** — uuid PK, todo_id (FK), name, position, timestamps
12. **checklist_items** — uuid PK, checklist_id (FK), content, is_checked (bool default false), position, timestamps
13. **labels** — uuid PK, workspace_id (FK), name, color, timestamps
14. **tags** — uuid PK, workspace_id (FK), name, timestamps
15. **todo_label** (pivot) — todo_id (FK), label_id (FK). Composite PK or unique constraint
16. **todo_tag** (pivot) — todo_id (FK), tag_id (FK). Composite PK or unique constraint
17. **comments** — uuid PK, todo_id (FK), user_id (FK), body (text), timestamps, soft_deletes
18. **reminders** — uuid PK, todo_id (FK), user_id (FK), reminded_at (timestamp), is_sent (bool default false), type (enum: email/in_app/browser), timestamps
19. **attachments** — uuid PK, todo_id (FK), user_id (FK), filename, path (storage path), mime_type, size (int bytes), timestamps
20. **activity_logs** — uuid PK, user_id (FK nullable), workspace_id (FK nullable), subject_type, subject_id, event (string: created/updated/completed/deleted/etc), properties (json nullable), timestamps
21. **user_preferences** — uuid PK, user_id (FK unique), timezone, language, date_format, time_format, theme (system/light/dark), default_view, start_page, notification_email (bool), notification_browser (bool), notification_in_app (bool), json config (nullable), timestamps
22. **notifications** — use Laravel's built-in table (uuid PK, type, notifiable_type, notifiable_id, data json, read_at nullable, timestamps)

**Indexes (critical for SQLite performance):**
- All FK columns
- `todos.workspace_id`, `todos.project_id`, `todos.status`, `todos.priority`, `todos.due_date`, `todos.assigned_to`, `todos.is_archived`, `todos.is_pinned`, `todos.is_favorite`, `todos.completed_at`, composite `[project_id, status]`, `[workspace_id, status]`
- `projects.workspace_id`
- `activity_logs.subject_type + subject_id`, `activity_logs.user_id`, `activity_logs.workspace_id`
- `workspace_members.workspace_id`, `workspace_members.user_id`
