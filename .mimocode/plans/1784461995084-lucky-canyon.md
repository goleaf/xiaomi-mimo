# Fix: /activity page — "plain JSON response" Inertia error

## Root Cause

`NotificationController` returns `JsonResponse` for all three endpoints, but these routes live inside the `auth`+`verified` middleware group in `routes/web.php` and are accessed via Inertia's `<Link>` component in the sidebar. Inertia expects an Inertia-compatible response (HTML page or redirect), not plain JSON.

When the user navigates to `/notifications` from the sidebar (or any page), Inertia receives the JSON payload and throws:

> All Inertia requests must receive a valid Inertia response, however a plain JSON response was received.

## Fix

**File: `app/Http/Controllers/NotificationController.php`**

1. **`index()`** — Change from `response()->json(...)` to `Inertia::render('notifications/Index', ['notifications' => $notifications])` so the notifications page renders as a proper Inertia page.

2. **`markRead()`** — Change from `response()->json(null, 204)` to `redirect()->back()` so the POST redirect is Inertia-compatible.

3. **`markAllRead()`** — Same: change from `response()->json(null, 204)` to `redirect()->back()`.

## Files to Modify
