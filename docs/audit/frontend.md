# Frontend Audit

## Scope And Inventory

The Vue application uses 24 Inertia pages, 8 layouts, 156 Vue components, 9 composables, 5 Pinia stores, 8 type modules, and 3 library modules. All pages use Vue 3 single-file components; the application bootstrap is `resources/js/app.ts`.

### Pages

- Root: `Dashboard.vue`.
- Authentication: `ConfirmPassword.vue`, `ForgotPassword.vue`, `Login.vue`, `Register.vue`, `ResetPassword.vue`, `TwoFactorChallenge.vue`, `VerifyEmail.vue`.
- Product: `activity/Index.vue`, `calendar/Index.vue`, `notifications/Index.vue`, `projects/Index.vue`, `projects/Show.vue`, `tasks/Index.vue`, `tasks/Show.vue`, `workspaces/Index.vue`.
- Settings: `Appearance.vue`, `Backup.vue`, `Export.vue`, `Members.vue`, `Notifications.vue`, `Preferences.vue`, `Profile.vue`, `Security.vue`.

### Layouts And Components

- Layouts: `AppLayout`, `AuthLayout`, `app/AppHeaderLayout`, `app/AppSidebarLayout`, three auth variants, and `settings/Layout`.
- Application shell/navigation/auth components: 28 root components including `AppSidebar`, `AppHeader`, `NavMain`, `NavUser`, passkey/2FA components, form helpers, and appearance controls.
- Feature components: dashboard `ProductivityChart`; project `ProjectCreateDialog`; shared `CommandPalette` and `EmptyState`; task `BoardView`, `BulkActions`, `RecurringBadge`, `TaskCreateDialog`, `TaskDetail`, `TaskFilterBar`, `TaskStats`; workspace `WorkspaceSwitcher`.
- Reka/shadcn-style primitives: alert (3), avatar (3), badge (1), breadcrumb (7), button (1), card (7), checkbox (1), collapsible (3), dialog (10), dropdown menu (14), input (1), input OTP (4), label (1), navigation menu (8), select (11), separator (1), sheet (9), sidebar (24), skeleton (1), sonner (1), spinner (1), and tooltip (4).

### Client Logic And Types

- Composables: `useAppearance`, `useAutosave`, `useBulkSelect`, `useCurrentUrl`, `useInitials`, `useKeyboard`, `useKeyboardShortcuts`, `useToast`, `useTwoFactorAuth`.
- Pinia: notification, project, todo, UI, and workspace stores.
- Types: API, auth, globals, aggregate index, models, navigation, UI, and Vue shims.
- Libraries: flash-to-toast bridging, custom route lookup, and class-name utilities.
- Route sources: generated `resources/js/actions` and `resources/js/routes` trees plus the custom `resources/js/lib/route.ts` global.

## Focused Task Experience Review

| Surface                                | Verified current behavior                                                                                 | Gap                                                                                                                               |
| -------------------------------------- | --------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `pages/tasks/Index.vue`                | Owns filters, selection, mutations, task opening, view buttons, rows, formatting, and dialog coordination | Too much page logic; no extracted filter/bulk/board/pagination/error architecture                                                 |
| `pages/tasks/Show.vue`                 | Full-page task detail with status changes, checklists, comments, and deletion                             | Duplicates detail behavior and uses one checklist-item input across all checklists                                                |
| `components/task/TaskDetail.vue`       | Custom fixed detail panel with inline mutations and local refs initialized from props                     | No task-change synchronization, focus trap, overlay, dialog semantics, escape/return-focus handling, or per-checklist input state |
| `components/task/BoardView.vue`        | Native `draggable`, drag events, optimistic local status copy, and direct mutation request                | Imports unused `@dnd-kit` APIs, lacks touch/keyboard alternative and robust rollback/order persistence                            |
| `components/task/TaskCreateDialog.vue` | Uses the existing Reka dialog wrapper and submits with Inertia                                            | Relies on hardcoded copy/global route helper; async failure feedback is incomplete                                                |
| `components/task/TaskFilterBar.vue`    | Separate filter UI exists                                                                                 | It is not imported by the task page; filtering is duplicated                                                                      |
| `components/task/BulkActions.vue`      | Complete/archive/delete controls exist                                                                    | It is not imported by the task page and does not provide the full required action set                                             |

## Current Tooling Evidence

The phase-0 baseline found 15 Vue/TypeScript errors, 88 ESLint errors, and 19 Prettier-unformatted resource files while the production Vite build succeeded. Prompt 1 does not modify the production frontend, so these remain unresolved audit inputs until the responsible implementation phases.

No component, dependency, generated route, or runtime frontend file was changed in this audit phase.
