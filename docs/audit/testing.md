## Test Strategy By Risk

1. Phase 2 starts with cross-workspace and role tests before changing authorization code.
2. Phase 3 adds route/API contract tests before deduplicating controllers.
3. Phase 4 establishes migration and query-count safeguards before schema/index/query changes.
4. Phases 5-10 add focused backend tests plus component/accessibility verification for each user workflow.
5. Phase 11 reruns the complete matrix against fresh, seeded, and representative populated SQLite databases and verifies backup/restore only in isolated test storage.
